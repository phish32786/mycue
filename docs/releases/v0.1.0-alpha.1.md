# MyCue 0.1.0-alpha.1

First public packaged build of MyCue for the XENEON EDGE hardware path.

## Highlights

- dedicated XENEON kiosk mode with hardware display detection
- HID touch seize, calibration, and recovery flow
- in-app settings on the main display
- bundled plugins for weather, Spotify, system stats, launcher, web widget, media gallery, and F1
- multi-page dashboard layout with on-device edit mode
- signed and notarized macOS distribution
- drag-to-Applications DMG packaging

## Notes

- Input Monitoring is still required for the touchscreen seizure path
- OpenF1 data is currently using completed sessions for stability
- the DMG is intended as the primary end-user install path

## Included artifacts

- `MyCue.dmg`
- `MyCue.zip`

## Known limits

- F1 live current-race mode is intentionally deferred until a proper realtime transport is added
- HID/Input Monitoring remains the most fragile part of the first-run setup
- some workflow filenames and helper script names still reflect earlier internal naming
